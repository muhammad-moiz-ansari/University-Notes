🔹 Slide 11 — Internet Inter-AS Routing: BGP

1️⃣ Slide Summary

Introduces BGP (Border Gateway Protocol) — the main protocol for routing between Autonomous Systems (ASes).

2️⃣ Detailed Explanation

BGP is called:

“The glue that holds the Internet together.”

BGP allows:

  • A subnet to announce its existence to the entire Internet

  • An AS to declare which destinations it can reach, and which paths it prefers

Two main parts:

  • eBGP (external BGP): exchange routes between ASes

  • iBGP (internal BGP): spread external routes inside the AS

Routers use BGP to:

  • Select “good” routes using multiple criteria

  • Enforce policy, e.g., which AS they want/don’t want to route traffic through

3️⃣ Diagram Explanation

None (text slide). Focus is on the conceptual idea of eBGP and iBGP.

4️⃣ Extra Context

BGP runs over TCP to ensure reliable message delivery.

Unlike OSPF (performance-focused), BGP is policy-focused.

5️⃣ Real-world Applications

Used by ISPs and large organizations to control global Internet routing.

6️⃣ Exam Questions

Q1. What is eBGP used for?
A: To exchange routing information between ASes.

Q2. Why is BGP policy-driven?
A: Because ASes want control over who uses their network.

🔹 Slide 12 — eBGP and iBGP Connections

1️⃣ Slide Summary

Shows how routers within and between ASes communicate using BGP.

2️⃣ Detailed Explanation

  • eBGP sessions occur between gateway routers of two ASes.

  • iBGP sessions occur inside the AS, allowing one gateway’s info to be shared with internal routers.

Every gateway router typically runs:

  • eBGP (to talk to other ASes)

  • iBGP (to share info internally)

3️⃣ Diagram Explanation

Shows AS1, AS2, AS3.

  • Solid lines between ASes = eBGP

  • Internal connections = iBGP mesh

The diagram highlights how reachability learned externally is spread internally.

4️⃣ Extra Context

In larger networks, a full iBGP mesh is hard; route reflectors or confederations are used.

5️⃣ Applications

Used inside large ISPs to ensure every router knows where external networks are.

6️⃣ Exam Questions

Q1. What does iBGP propagate?
A: Reachability information learned from eBGP.

🔹 Slide 13 — BGP Basics

1️⃣ Slide Summary

Explains how BGP routers communicate path information.

2️⃣ Detailed Explanation

Example:

  • AS3 router 3a advertises path AS3, X to AS2 router 2c.

  • This means “I can reach X, and I promise to forward traffic to X.”

BGP Session:

  • Runs over a TCP connection

  • Two routers (called BGP peers) exchange:

    • Path updates

    • Keepalives

    • Notifications

BGP is a path vector protocol (extension of distance vector).

Each update includes:

  • Destination prefix

  • Entire AS-path

3️⃣ Diagram Explanation

Shows AS3 → AS2 → AS1 router interactions.

4️⃣ Extra Context

The inclusion of the full AS-path prevents routing loops — routers detect if their own AS is in the path.

5️⃣ Real-world Applications

Ensures stable routing over the global Internet.

6️⃣ Exam Questions

Q1. Why does BGP include the full AS path in advertisements?
A: To help avoid routing loops.

🔹 Slide 14 — Path Attributes and BGP Routes

1️⃣ Slide Summary

Explains key attributes inside a BGP route advertisement.

2️⃣ Detailed Explanation

Every BGP route = Prefix + Attributes

Two important attributes:

  1. AS-PATH

    • Sequence of ASes that the route passed through.

    • Used for loop prevention + shortest-path preference.

  2. NEXT-HOP

    • IP address of the router in the next AS.

    • Tells the receiving AS where to forward packets.

Policy-based routing:

  • Import policy: decide whether to accept a route

  • Export policy: decide whether to advertise it further

3️⃣ Diagram Explanation

Shows AS1, AS2, AS3; NEXT-HOP points toward gateway routers.

4️⃣ Extra Context

Policies allow ISPs to decide:

  • Which customers’ routes to advertise

  • Which providers to prefer

  • Which paths to avoid (economic or political reasons)

5️⃣ Applications

ISPs make millions of dollars by controlling traffic paths (e.g., peering agreements).

6️⃣ Exam Questions

Q1. What is the NEXT-HOP attribute used for?
A: To indicate the router where packets should be forwarded next.

🔹 Slide 15 — BGP Path Advertisement

1️⃣ Slide Summary

Shows how AS2 receives and propagates BGP paths.

2️⃣ Detailed Explanation

Example:

  1. AS3 → AS2: Advertises AS3, X

  2. AS2's router 2c accepts it (policy permits)

  3. 2c spreads this internally using iBGP

  4. Another gateway 2a advertises AS2, AS3, X to AS1

3️⃣ Diagram Explanation

The arrows show path propagation:

  • eBGP: AS3 → AS2, AS2 → AS1

  • iBGP: inside AS2

4️⃣ Extra Context

Each AS prepends its ASN to the path before forwarding.

5️⃣ Real-world Applications

Useful for understanding why Internet paths sometimes appear longer — due to policy constraints.

6️⃣ Exam Questions

Q1. What happens to the AS path when a route crosses a new AS?
A: The receiving AS prepends its ASN to the AS-PATH attribute.

🔹 Slide 16 — BGP Path Advertisement (More)

1️⃣ Slide Summary

Shows how AS1 learns multiple paths to the same destination and chooses one.

2️⃣ Detailed Explanation

AS1 gateway router 1c learns:

  • Path 1: AS2, AS3, X

  • Path 2: AS3, X

It compares them using policy and selects AS3, X (shorter path).

Then 1c advertises the chosen path to internal routers using iBGP.

3️⃣ Diagram Explanation

Displays multiple incoming path announcements to AS1.

4️⃣ Extra Context

Routers evaluate alternative routes based on:

  • Local preference

  • AS-path length

  • Hot-potato routing

  • Etc.

5️⃣ Applications

Routers pick paths based on business relationships (customer > peer > provider).

6️⃣ Exam Questions

Q1. Why might AS1 choose AS3,X over AS2,AS3,X?
A: It has a shorter AS-PATH.

🔹 Slide 17 — BGP Messages

1️⃣ Slide Summary

Lists the four types of BGP messages.

2️⃣ Detailed Explanation

BGP uses TCP for reliable communication. Messages:

  1. OPEN

    • Starts a BGP session

    • Includes ASN, authentication, etc.

  2. UPDATE

    • Advertises new routes

    • Withdraws old routes

  3. KEEPALIVE

    • Confirms OPEN

    • Ensures connection stays active when no UPDATEs are needed

  4. NOTIFICATION

    • Sent when errors occur

    • Also closes the session

3️⃣ Diagram Explanation

None required; list slide.

4️⃣ Extra Context

Most of BGP’s communication is “silent” via KEEPALIVE messages to maintain TCP state.

5️⃣ Applications

Used by ISPs to maintain stable inter-AS routing sessions.

6️⃣ Exam Questions

Q1. Which BGP message is used to close a connection?
A: NOTIFICATION.

🔹 Slide 18 — BGP Path Advertisement & Intra-AS Forwarding

1️⃣ Slide Summary

Explains how routers inside AS1 use OSPF to reach the gateway that BGP selected.

2️⃣ Detailed Explanation

Steps:

  • Internal routers (1a, 1b, 1d) learn the external path from 1c through iBGP.

  • They know: “To reach X, forward traffic to router 1c.”

  • To reach 1c, they use OSPF to compute the shortest path internally.

Thus routing to X = OSPF path to 1c → BGP path out of AS.

3️⃣ Diagram Explanation

Shows internal router tables:

  • Destination X → interface leading to 1c

  • Destination 1c determined via OSPF

4️⃣ Extra Context

This is called forwarding based on next-hop resolution.

5️⃣ Applications

Every AS uses this combination:
OSPF for internal routing + BGP for external routing.

6️⃣ Exam Questions

Q1. Why is OSPF still needed when using BGP?
A: OSPF provides internal routes needed to reach the BGP-selected gateway router.

🔹 Slide 19 — Alternate Intra-AS Paths

1️⃣ Slide Summary

Shows another scenario where internal routers choose different interfaces (based on OSPF costs) to reach the same BGP gateway.

2️⃣ Detailed Explanation

Different internal routers may choose different internal paths to reach the same gateway, depending on link costs.

Example:

Even though both ultimately exit through 1c to reach X.

Forwarding tables show:

  • BGP decides which gateway

5️⃣ Applications

Useful for load distribution inside AS.

6️⃣ Exam Questions

Q1. Does BGP compute internal router paths?
A: No, internal paths are computed by OSPF (or another IGP).

🔹 Slide 20 — Why Different Intra- and Inter-AS Routing?

1️⃣ Slide Summary

Compares goals of intra-AS vs inter-AS routing.

2️⃣ Detailed Explanation

Inter-AS routing priorities:

Intra-AS routing priorities:

Simple conceptual slide.

Performance vs. Policy distinction dominates real ISP operations.

🔹 Slide 21 — Hot Potato Routing

1️⃣ Slide Summary

Explains a key BGP routing decision: Hot Potato Routing, where routers choose the gateway that minimizes intra-AS cost, not the shortest AS-path.

2️⃣ Detailed Explanation

Suppose router 2d in AS2 learns two possible exit gateways for destination X:

  • via 2a

  • via 2c

Even if 2c → (AS3 → X) is shorter externally, router 2d may pick 2a because:

Hot potato routing = choose the closest (cheapest) internal gateway.

Reason:

  • ISPs prefer to push traffic out of their network as quickly as possible, minimizing the use of their own internal resources.

3️⃣ Diagram Explanation

The diagram shows:

  • OSPF link weights inside AS2 (e.g., 2d → 2a = cost 1, 2d → 2c = cost 3)

  • Externally, 2c has a better AS-path, but 2d still chooses 2a.

4️⃣ Extra Context

Hot potato routing can lead to:

  • Suboptimal global performance

  • Traffic imbalances

  • Unexpected routes between ASes

But it is widely used.

5️⃣ Real-world Applications

Used in major ISPs to reduce internal infrastructure load.

6️⃣ Exam Questions

Q1. What is hot potato routing?
A: Choosing the nearest internal exit gateway, regardless of external AS-path length.

🔹 Slide 22 — BGP Achieving Policy via Advertisements (Example 1)

1️⃣ Slide Summary

Shows how ISPs can enforce routing policies by selectively advertising paths.

2️⃣ Detailed Explanation

Scenario:

  • A, B, C are provider networks.

  • w is a customer network of A.

  • A advertises route to w → to both B and C.

  • B receives route A–w but chooses not to advertise it to C, because:

B gets no revenue for routing traffic between C → A → w.

So C does not learn route C–B–A–w and must use C–A directly.

3️⃣ Diagram Explanation

The diagram shows:

  • Provider and customer relationships

  • Missing advertisement from B → C

4️⃣ Extra Context

This is called no-transit policy:
ISPs do not want to carry traffic between two networks that are not their customers.

5️⃣ Real-world Applications

This is why the Internet’s topology is shaped by business relationships, not shortest paths.

6️⃣ Exam Questions

Q1. Why would an ISP choose not to advertise a route?
A: To avoid providing free transit between other ISPs.

🔹 Slide 23 — BGP Achieving Policy via Advertisements (Example 2)

1️⃣ Slide Summary

Adds complexity by introducing a dual-homed customer (x), which connects to two providers.

2️⃣ Detailed Explanation

Scenario:

  • Providers: A, B, C

  • Customers: w, x, y

  • x is connected to both A and B (dual-homed)

x’s policy:

x does not want to carry transit traffic for B → C.

Therefore:

  • x does not advertise a route to C to B

  • This prevents B from routing through x to reach C

3️⃣ Diagram Explanation

Shows x connecting to both A and B, but refusing to forward C-bound paths.

4️⃣ Extra Context

Policies in BGP are often enforced by filtering advertisements.

5️⃣ Real-world Applications

Enterprise networks often dual-home to two ISPs but do not want to act as routers between them.

6️⃣ Exam Questions

Q1. What is the effect of a dual-homed network not advertising routes to one provider?
A: It prevents that provider from sending transit traffic through it.

🔹 Slide 24 — BGP Route Selection

1️⃣ Slide Summary

Lists the main criteria used by BGP to select best routes.

2️⃣ Detailed Explanation

🌟 1. Full Chapter Summary (Detailed Revision Notes)

A. Intra-AS Routing (OSPF)

  • OSPF = Link-state routing protocol used inside an Autonomous System.

  • Each router floods LSAs to all other routers in the AS.

  • All routers maintain the same topology map.

  • Runs Dijkstra’s SPF algorithm to compute shortest paths.

  • Supports multiple equal-cost paths (ECMP).

  • Hierarchical structure:

    • Backbone Area (Area 0)

    • Other areas connect to Area 0

  • Advantages:

    • Fast convergence

    • Supports authentication

    • Supports hierarchical design

B. Inter-AS Routing (BGP)

  • BGP = Border Gateway Protocol

  • Used between ISPs (ASes) across the Internet.

  • Path-vector protocol — advertises full AS-PATH.

  • BGP routers exchange prefix reachability info.

  • Policies influence routing more than shortest paths.

  • Two types of sessions:

    • eBGP (between ASes)

    • iBGP (inside an AS)

BGP Route Selection (in order of preference):

  1. Local Preference

  2. Shortest AS-PATH

  3. Origin type (IGP > EGP > incomplete)

  4. MED

  5. Hot potato routing (closest exit)

  6. Lowest router ID

Hot Potato Routing

  • Choose the closest internal exit even if external path is longer.

  • Goal: minimize internal transit cost of ISP.

Policy Control

ISPs may:

  • Not advertise certain routes

  • Not carry transit traffic for another ISP

  • Prefer customer routes over peer/provider routes

C. SDN: Software-Defined Networking

Key Principles:

  1. Generalized, flow-based forwarding

  2. Separation of control plane and data plane

  3. Control logic moved to a central controller

  4. Programmability of control applications

Controller → Switch:

🌟 3. Long Theory Questions + Perfect Answers

Q1. Explain the differences between OSPF and BGP.

Answer:
OSPF is an intra-AS link-state routing protocol used within organizations. It floods LSAs, builds a complete topology map, and computes shortest paths using Dijkstra. It aims for fast convergence and stable internal routing.

BGP is the inter-AS policy-based routing protocol used across the Internet. Instead of shortest paths, it uses AS-PATH, Local Preference, and policies to choose routes. BGP routers do not maintain full network topology and are influenced heavily by business relationships.

Q2. Describe the key principles of Software-Defined Networking (SDN).

Answer:
SDN is based on four principles:

  1. Flow-based forwarding: Match on multiple fields, not just destination IP.

  2. Separation of control & data planes: Switches only forward, controller computes logic.

  3. Centralized control: Controller maintains global state and installs flow rules.

  4. Programmability: Applications implement routing, security, and management functions.

Q3. Explain how Traceroute works, step-by-step.

Answer:
Traceroute sends packets with TTL values starting at 1 and increasing by 1 for each probe.
Each router that decrements TTL to zero sends ICMP Time Exceeded back.
This reveals the path hop-by-hop.
When packets reach the destination, an ICMP Port Unreachable is returned, marking the end.

Q4. What is hot potato routing and why is it used?

Answer:
Hot potato routing chooses the closest internal exit router to minimize internal transit cost. ISPs push traffic out of their network as quickly as possible to reduce load, even if this results in a longer path over the Internet.

Q5. Compare traditional routing with SDN-based routing.

Answer:
Traditional routing uses distributed algorithms (OSPF, BGP) where each router computes its own forwarding table. It is limited to destination-based forwarding.

SDN centralizes routing logic. The controller has a global view, computes optimal rules, and installs them in switches. SDN supports generalized forwarding, policy enforcement, load balancing, and fast reconfiguration during failures.
